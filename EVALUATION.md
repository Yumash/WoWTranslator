# WoW Chat Translator — Финальная оценка

## 1. Задача

Инструмент для автоматического перевода текстовых сообщений в чате World of Warcraft для мультиязыковых групп. Личный проект с возможностью публичного распространения.

## 2. Почему нельзя сделать "просто аддон"

WoW аддоны работают в изолированной Lua-среде:
- **Нет HTTP-запросов** — невозможно вызвать API переводчика
- **Нет доступа к файлам** — нельзя читать/писать произвольные файлы
- **Нет запуска процессов** — os.execute() заблокирован

Все существующие "переводчики" на CurseForge (WoW Translator, ChatTranslator EN-ES, TranslateCN) — это словари из 500-1000 фраз. Перевести произвольное предложение они не могут.

## 3. Ограничения Midnight (Patch 12.0)

Blizzard ввёл систему **Secret Values** — чат становится нечитаемым для аддонов в определённых ситуациях:

| Ситуация | Чат доступен аддонам? |
|----------|----------------------|
| Открытый мир, города, квесты | Да |
| В инстансе, между пулами | Да |
| Между боссами в рейде | Да |
| Гильд-чат, личные сообщения | Да |
| **M+ ключ (пока таймер идёт)** | **Нет** (Secret Values) |
| **PvP матч** | **Нет** |
| **Бой с боссом (encounter)** | **Нет** |

**Важно:** это ограничение Lua API для аддонов. Лог-файл `WoWChatLog.txt` пишется самим клиентом WoW на диск — Secret Values на него не влияют (требует подтверждения на живом клиенте).

## 4. Решение: Companion App + мини-аддон

### Архитектура

```
┌──────────────────────────────────────┐
│          World of Warcraft           │
│                                      │
│  [Мини-аддон ChatTranslatorHelper]   │
│   • /chatlog — включает запись       │
│   • Настройки: мой язык, каналы      │
│   • ~100-200 строк Lua               │
└──────────┬───────────────────────────┘
           │
           │  WoWChatLog.txt
           │  (WoW пишет непрерывно)
           ▼
┌──────────────────────────────────────┐
│     Companion App (Python .exe)      │
│                                      │
│  File Watcher → новые строки в логе  │
│  Parser → извлекает автора и текст   │
│  Lang Detect → определяет язык       │
│  Translator → DeepL Free API         │
│  Cache → не переводит повторы        │
│                                      │
│  ┌────────────────────────────────┐  │
│  │  Overlay / отдельное окно      │  │
│  │  поверх WoW с переводами      │  │
│  │  [Spieler]: Who tanks boss?   │  │
│  │  [김민수]: Let's go left       │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### Поток данных

```
Игрок (немец) пишет в party: "Wer tankt den Boss?"
    ↓
WoW Chat Log: [Party] [Spieler-Antonidas]: Wer tankt den Boss?
    ↓ (~200ms — file watcher)
Companion App: новая строка → парсинг
    ↓ (~100ms — langdetect)
Язык: DE (немецкий), мой язык: RU → нужен перевод
    ↓ (~300ms — DeepL API)
Перевод: "Кто танкует босса?"
    ↓
Overlay: "[Spieler]: Кто танкует босса?"

Общая задержка: ~1-2 секунды
```

### Соответствие ToS Blizzard

| Аспект | Статус |
|--------|--------|
| Companion app читает файл на диске | **Разрешено** (Wowhead Client, CurseForge App, WagoApp делают то же) |
| Overlay поверх окна | **Разрешено** (Discord Overlay работает так же) |
| Аддон включает /chatlog | **Разрешено** (штатная функция WoW) |
| Нет инъекции в процесс WoW | **Разрешено** |
| Нет автоматизации игровых действий | **Разрешено** |
| Нет перехвата сетевого трафика | **Разрешено** |

## 5. Стек технологий

### Companion App

| Компонент | Технология | Почему |
|-----------|------------|--------|
| Язык | Python 3.12 | Быстрая разработка, экосистема |
| GUI | PyQt6 | Нативный вид, прозрачные окна, always-on-top |
| File Watcher | watchdog | Стандарт для мониторинга файлов |
| Language Detection | lingua-py | Быстрый, точный, оффлайн |
| Translation API | DeepL Free (основной) | Лучшее качество, 500K символов/мес бесплатно |
| Translation Fallback | Google Translate (googletrans) | Запасной, без лимитов |
| Кэш | dict in-memory + SQLite | Повторяющиеся фразы не переводятся заново |
| Сборка | PyInstaller | Один .exe без зависимостей |

### WoW Аддон

| Компонент | Технология |
|-----------|------------|
| Язык | Lua 5.1 (WoW) |
| Функциональность | Включение /chatlog при загрузке |
| Настройки | SavedVariables (язык, каналы) |
| Slash-команда | /wct — настройки |

## 6. Функциональность

### MVP (первая версия)

- Мониторинг WoWChatLog.txt в реальном времени
- Автоопределение языка входящих сообщений
- Перевод на выбранный язык (RU/EN/DE/FR/ES)
- Overlay-окно поверх WoW (прозрачное, перетаскиваемое)
- Фильтр каналов: Party, Raid, Guild, Say, Whisper
- Игнорирование своих сообщений (не переводить себя)
- Кэш переводов
- Системный трей (свернуть, настройки)
- Мини-аддон: авто-включение /chatlog

### Дальнейшее развитие (v2)

- Пользовательский словарь игрового сленга (поверх API)
- Быстрый ответ: набрал текст → перевёл → скопировал в буфер → Ctrl+V в WoW
- Горячие клавиши (вкл/выкл, сменить язык)
- История переводов с поиском
- Поддержка нескольких API (DeepL / Google / Ollama)
- Автозапуск с WoW (детект процесса Wow.exe)

## 7. Стоимость эксплуатации

Для среднего игрока (~1000 сообщений/день, ~50 символов/сообщение):

| Сервис | Стоимость | Хватит на |
|--------|-----------|-----------|
| DeepL Free | Бесплатно | ~10K сообщений/мес |
| DeepL Pro | €5.49/мес | Безлимит |
| googletrans | Бесплатно | Без лимитов (неофициальный) |
| Ollama (локальный) | Бесплатно | Без лимитов, нужна GPU |

**Для личного использования DeepL Free хватит с запасом.**

## 8. Этапы работ

### Этап 1: Ядро Companion App (6-8 часов)
- File watcher для WoWChatLog.txt
- Парсер формата лога (regex)
- Определение языка (lingua-py)
- Интеграция DeepL Free API
- Кэш переводов (in-memory)
- CLI-режим для тестирования

### Этап 2: GUI + Overlay (5-6 часов)
- PyQt6 окно: список переводов, прокрутка
- Always-on-top + прозрачность
- Перетаскивание, изменение размера
- Настройки: язык, каналы, API ключ, путь к WoW
- Системный трей
- Авто-определение пути к WoWChatLog.txt

### Этап 3: WoW Аддон (2-3 часа)
- TOC-файл для Midnight (12.0)
- Авто-включение /chatlog при старте
- Slash-команда /wct
- SavedVariables для настроек

### Этап 4: Полировка и сборка (3-4 часа)
- PyInstaller → .exe
- Тестирование на живом клиенте
- Edge cases: ссылки, {rt} маркеры, эмодзи, пустые сообщения
- README с инструкцией установки
- Иконка, трей-меню

### Итого: 16-21 час
