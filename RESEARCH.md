# Технический анализ: Аддон автоперевода чата WoW

## 1. Среда исполнения аддонов WoW

### Lua Sandbox
WoW аддоны работают в изолированной Lua-среде с жёсткими ограничениями:
- **Нет сетевого доступа** — невозможно делать HTTP-запросы, открывать сокеты
- **Нет доступа к файловой системе** — нельзя читать/писать произвольные файлы
- **Нет запуска внешних процессов** — os.execute() и подобные заблокированы
- **Ограниченная стандартная библиотека** — многие Lua-функции удалены

### Единственные каналы ввода/вывода аддона:
1. **WoW API** — чтение игровых данных через разрешённые функции
2. **SavedVariables** — файлы данных, записываемые при выходе/перезагрузке UI
3. **SendAddonMessage()** — межаддонная коммуникация через игровые каналы
4. **Chat API** — чтение и отправка сообщений в чат

### Изменения в Midnight (Patch 12.0, январь 2026)

#### Secret Values
Новая система безопасности — "чёрные ящики" для данных:
- Аддоны получают Secret Values, но не могут прочитать содержимое
- Невозможно сравнивать, выполнять арифметику
- Активируются в контексте боевых действий

#### Ограничения чата в инстансах
- **Combat Log конвертирован в KStrings** — парсинг заблокирован
- **Сообщения чата → Secret Values** в определённых контекстах:
  - Mythic+ ключи (в процессе)
  - PvP матчи (в процессе)
  - Активные энкаунтеры (бой с боссом)
- **Вне этих ситуаций чат читается нормально**
- Blizzard закрывает workaround'ы для обхода

#### Позитивные изменения
- strsplit, strjoin, strconcat возвращены в restricted environment
- Ослаблены ограничения для UI-кастомизации (castbars, unitframes)
- Новые API для форматирования (SecondsFormatter)

## 2. Chat API — что доступно

### Чтение сообщений
```lua
-- Регистрация на события чата
frame:RegisterEvent("CHAT_MSG_PARTY")
frame:RegisterEvent("CHAT_MSG_PARTY_LEADER")
frame:RegisterEvent("CHAT_MSG_RAID")
frame:RegisterEvent("CHAT_MSG_RAID_LEADER")
frame:RegisterEvent("CHAT_MSG_RAID_WARNING")
frame:RegisterEvent("CHAT_MSG_SAY")
frame:RegisterEvent("CHAT_MSG_YELL")
frame:RegisterEvent("CHAT_MSG_WHISPER")
frame:RegisterEvent("CHAT_MSG_GUILD")
frame:RegisterEvent("CHAT_MSG_INSTANCE_CHAT")

-- Обработчик
frame:SetScript("OnEvent", function(self, event, msg, author, ...)
    -- msg = текст сообщения (string, НО может быть Secret в инстансах)
    -- author = имя отправителя
end)
```

### Отправка сообщений
```lua
SendChatMessage("текст", "PARTY") -- в группу
SendChatMessage("текст", "RAID")  -- в рейд
SendChatMessage("текст", "WHISPER", nil, "Имя-Сервер") -- в личку
```

### Межаддонная коммуникация
```lua
C_ChatInfo.SendAddonMessage("PREFIX", data, "PARTY")
-- Каналы: PARTY, RAID, GUILD, BATTLEGROUND
-- НЕ работает: WHISPER, SAY, YELL
-- Ограничение: ~250 символов на сообщение
```

### ChatFrame фильтры
```lua
ChatFrame_AddMessageEventFilter("CHAT_MSG_PARTY", function(self, event, msg, ...)
    -- Можно модифицировать отображаемое сообщение
    -- return false, modifiedMsg, ...
end)
```

## 3. Существующие аддоны-переводчики

### WoW Translator (CurseForge)
- **Метод:** встроенный словарь сленга и акронимов
- **Подход:** подстановка известных сокращений (brb, lfg, oom, etc.)
- **Языки:** EN, ES, FR, DE, IT, PT, RU, KO, ZH
- **Ограничение:** НЕ полноценный перевод, только словарь терминов

### ChatTranslator EN-ES
- **Метод:** встроенный словарь для англо-испанской пары
- **Подход:** замена слов/фраз по таблице
- **Версия:** v1.5 (декабрь 2024)

### Polyglot
- **Метод:** peer-to-peer через SendAddonMessage
- **Подход:** игроки, понимающие язык, ретранслируют перевод другим
- **Каналы:** Party, Raid, Guild, BG (НЕ Say/Yell/Whisper)
- **Проблема:** нет защиты от ложных переводов, нужен носитель языка в группе

### TranslateCN
- **Метод:** словарь CN → EN для игрового сленга
- **Подход:** подстановка китайских терминов

### Dialect Chat
- **Метод:** чисто Lua, локальное хранение, фэнтези-диалекты
- **Подход:** развлекательный, не реальный перевод

## 4. Анализ подходов к реализации

### Подход A: Встроенный словарь (чисто аддон)

**Архитектура:**
```
[Чат WoW] → [Аддон: словарь фраз] → [Отображение перевода в ChatFrame]
```

**Плюсы:**
- Не нарушает ToS
- Нулевая задержка
- Работает оффлайн и в инстансах (где текст не Secret)
- Простая установка (только аддон)

**Минусы:**
- Качество перевода ОЧЕНЬ низкое — только фразы/шаблоны
- Невозможно перевести произвольный текст
- Огромный размер словаря для нормального покрытия (десятки МБ Lua-таблиц)
- Нужно поддерживать для каждой языковой пары

**Оценка:** 2/10 по качеству перевода

### Подход B: Companion App + SavedVariables

**Архитектура:**
```
[Чат WoW] → [Аддон записывает в SavedVariables] → /reload UI/
→ [Companion App читает .lua файл] → [Google Translate API] →
→ [App пишет перевод в .lua файл] → /reload UI/ → [Аддон показывает]
```

**Плюсы:**
- Качественный перевод (API/LLM)
- Формально не нарушает ToS (companion apps разрешены — Wowhead, WoWUp)

**Минусы:**
- **КРИТИЧЕСКИЙ:** SavedVariables записываются только при /reload или выходе из игры
- Задержка: минимум 5-10 секунд на цикл (reload → перевод → reload)
- /reload в бою невозможен
- Непрактично для реального общения

**Оценка:** 3/10 по юзабилити

### Подход C: Companion App + Chat Log файл

**Архитектура:**
```
[WoW пишет лог чата] → [Companion App мониторит файл] →
→ [Translation API] → [Overlay поверх игры / отдельное окно]
```

**Плюсы:**
- Качественный перевод
- Почти реальное время (WoW Chat Log пишется непрерывно)
- Не требует reload

**Минусы:**
- WoW ChatLog по умолчанию выключен, нужно включать
- Перевод показывается в отдельном окне (не в игровом чате)
- Нужна установка отдельного приложения
- Задержка ~1-3 секунды (чтение лога + API вызов)
- Нет обратной интеграции — ответить переведённым текстом нельзя автоматически

**Оценка:** 6/10 — работоспособно, но неудобно

### Подход D: Overlay-приложение с OCR

**Архитектура:**
```
[Экран WoW] → [OCR распознаёт текст чата] → [Translation API] →
→ [Overlay поверх окна чата с переводами]
```

**Плюсы:**
- Работает с ЛЮБЫМ текстом (не зависит от API аддонов)
- Не нужен аддон вообще
- Обходит все ограничения Secret Values
- Качественный перевод

**Минусы:**
- Сложная реализация OCR для игрового шрифта
- Задержка 2-5 секунд (OCR + перевод)
- Высокая нагрузка на CPU/GPU
- Может конфликтовать с античитом (overlay)
- **РИСК ToS:** Blizzard может рассматривать screen-reading как ботоводство

**Оценка:** 5/10 — технически мощно, но рискованно

### Подход E: Гибрид (аддон + companion app через файл-обмен)

**Архитектура:**
```
[Чат WoW] → [Аддон пишет в custom лог-файл через SavedVariables хак]
→ [Companion App мониторит] → [Translation API] →
→ [App пишет в другой файл] → [Аддон читает при следующем тике]
```

**Проблема:** Аддон НЕ может читать произвольные файлы. SavedVariables загружаются только при /reload. Этот подход нерабочий без reload.

### Подход F: Аддон + Discord Bot / Telegram Bot

**Архитектура:**
```
[Аддон показывает текст] → [Игрок копирует / видит] →
→ [Discord бот переводит в реальном времени в голосовом/текстовом канале]
```

Или через WoW → Discord интеграцию (сторонние инструменты).

**Плюсы:**
- Discord-боты с переводом уже существуют
- Не нарушает ToS

**Минусы:**
- Ручной процесс или нужна интеграция
- Не в игровом чате

## 5. Ограничения ToS / EULA

### Разрешено:
- Аддоны с встроенными словарями
- Companion apps, читающие SavedVariables и логи (Wowhead Client, CurseForge app)
- Overlay-программы типа Discord overlay

### Серая зона:
- Screen reading / OCR (может быть расценено как automation)
- Автоматическая модификация отправляемых сообщений

### Запрещено:
- Инъекция кода в процесс WoW
- Автоматизация игровых действий
- Перехват сетевого трафика WoW

## 6. Стоимость API переводчиков (для Companion App)

| Сервис | Стоимость | Качество | Скорость |
|--------|-----------|----------|----------|
| Google Translate API | $20/1M символов | Хорошее | ~200ms |
| DeepL API | $25/1M символов (Free: 500K/мес) | Отличное | ~300ms |
| LibreTranslate (self-hosted) | Бесплатно | Среднее | ~500ms |
| OpenAI GPT-4o-mini | $0.15/1M input tokens | Хорошее+ | ~500ms |
| Claude Haiku | $0.25/1M input tokens | Хорошее+ | ~400ms |
| Ollama (локальный LLM) | Бесплатно | Среднее-Хорошее | ~1-3s |

Для чата в WoW (короткие фразы, ~50 символов):
- ~20 символов = ~5 токенов
- 1000 сообщений/час ≈ 5000 токенов ≈ $0.001 (GPT-4o-mini)
- Практически бесплатно для любого API

## 7. Выводы

### Реалистичные варианты:

1. **Словарный аддон** — быстро и просто, но бесполезен для реального перевода
2. **Companion App + Chat Log** — лучший баланс качества и совместимости с ToS
3. **Overlay с OCR** — максимальная функциональность, но риски ToS
4. **Discord/Telegram интеграция** — если группа уже использует Discord
